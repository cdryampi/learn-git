
# Más Ejercicios de Git por Temas

## 1. Configuración de Git

- Configurar el nombre de usuario y el correo electrónico globalmente y por repositorio con `git config`.
- Verificar las configuraciones actuales y cambiarlas si es necesario.
- Personalizar un alias para un comando frecuente (ej. `git log` con una vista simplificada).

## 2. Inicialización y Primeros Pasos

- Inicializar un repositorio vacío con git init y verificar el contenido del directorio `.git`.
- Añadir varios archivos al área de staging con `git add` y confirmar los cambios con `git commit`.
- Crear un historial inicial con al menos tres commits y visualizarlo con `git log`.

## 3. Trabajo con Archivos

- Modificar un archivo, añadirlo al área de staging, y deshacer los cambios antes del commit con `git reset`.
- Usar `git rm` para eliminar un archivo del repositorio y registrar el cambio.
- Ignorar archivos o carpetas específicas creando un archivo .gitignore y validarlo con `git status`.

## 4. Exploración del Historial

- Revisar el historial de cambios con `git log` utilizando diferentes opciones para filtrar resultados (por ejemplo, `--oneline` o `--author`).
- Ver las diferencias entre el último commit y el área de trabajo actual con `git diff`.
- Usar `git blame` para identificar qué cambios se hicieron en una línea específica de un archivo.

## 5. Trabajo con Ramas

- Crear y cambiar entre ramas con `git branch` y `git checkout`.
- Fusionar una rama secundaria a la rama principal con `git merge`.
- Resolver un conflicto generado al intentar fusionar dos ramas que contienen cambios en la misma línea de un archivo.

## 6. Correcciones y Ajustes

- Modificar el mensaje del último commit usando `git commit --amend`.
- Restaurar un archivo al estado de un commit anterior con `git checkout`.
- Crear un commit que revierta los cambios realizados en un commit específico con `git revert`.

## 7. Análisis de Cambios

- Generar un parche de cambios con `git diff` y exportarlo a un archivo.
- Usar `git show` para explorar detalles de un commit específico.

## 8. Etiquetado

- Crear etiquetas ligeras y anotadas con `git tag`.
- Listar todas las etiquetas existentes en el repositorio y explorar su información con `git show <tag>`.

## 9. Limpieza

- Eliminar archivos sin seguimiento del repositorio usando `git clean` (con precaución).
- Configurar y utilizar `git stash` para guardar cambios temporales antes de cambiar de rama.

## 10. Optimización y Configuración Avanzada

- Cambiar la estrategia de fusión predeterminada al usar `git merge`.
- Usar `git reflog` para recuperar commits eliminados accidentalmente.
